This diagram shows three bars subdivided into startup events along a horizontal timeline.

The first bar is the default case and is divided into three boxes. The first section of the bar shows a screenshot of a progress bar, which represents the application download stage. The second section in the bar represents the security warning for the signed application. The third section of the bar represents application initialization.

The second bar is the case in which the certificate is embedded in the deployment descriptor. The entire bar is only about two thirds as long as the first bar. The first section of the second bar shows the security warning and the application download starting concurrently, so the security warning can be completed while the application is being downloaded. The second section of the bar shows application initialization.

The third bar is the same as the second bar, except instead of application initialization in the last section, there is an error dialog box, which also completes sooner than the sequence shown in the default case.
